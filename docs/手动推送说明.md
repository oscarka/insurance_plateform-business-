## 推送方式

### 方式1：使用推送脚本（推荐）

已创建 `push-to-github.sh` 脚本，会自动尝试多种方式：

```bash
cd /Users/cc/insuranceplateform
./push-to-github.sh
```

### 方式2：手动在终端执行

打开终端，执行：

```bash
cd /Users/cc/insuranceplateform

# 尝试HTTPS方式
git remote set-url origin https://github.com/oscarka/insurance_plateform-business-.git
git push origin main

# 如果HTTPS不行，尝试SSH（443端口，通常更稳定）
git remote set-url origin git@ssh.github.com:oscarka/insurance_plateform-business-.git
GIT_SSH_COMMAND="ssh -p 443" git push origin main
```

### 方式3：使用GitHub Desktop

1. 打开GitHub Desktop
2. 选择仓库
3. 点击 "Push origin" 按钮

### 方式4：使用VS Code

1. 在VS Code中打开项目
2. 点击左侧源代码管理图标
3. 点击 "..." → "推送"

### 方式5：稍后重试

如果网络暂时有问题，可以：
- 稍后网络恢复时再push
- 代码已安全保存在本地，不会丢失
- 可以先进行Cloudflare和Railway的配置，等网络好了再push

## 如果推送失败

代码已经安全保存在本地，可以：
1. 检查网络连接
2. 检查GitHub认证（可能需要重新输入密码或token）
3. 稍后重试
4. 使用其他方式（GitHub Desktop等）

## 重要提示

即使暂时无法push到GitHub，你也可以：
- ✅ 先在Railway部署后端（可以手动上传代码或使用Railway CLI）

- ✅ 代码已保存在本地，不会丢失

等网络恢复后，再push到GitHub即可。
