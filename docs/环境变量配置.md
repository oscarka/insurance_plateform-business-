# 环境变量配置说明

## 一、后端环境变量

在 `backend/` 目录创建 `.env` 文件：

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=insurance_platform

# 服务器配置
PORT=3000
NODE_ENV=development

# CORS配置（生产环境）
# 多个域名用逗号分隔
ALLOWED_ORIGINS=https://your-user-frontend.pages.dev,https://your-admin-frontend.pages.dev
```

### Railway部署时的环境变量

在Railway项目设置中添加：

```env
DB_HOST=${{MySQL.MYSQLHOST}}
DB_PORT=3306
DB_USER=${{MySQL.MYSQLUSER}}
DB_PASSWORD=${{MySQL.MYSQLPASSWORD}}
DB_NAME=${{MySQL.MYSQLDATABASE}}
PORT=3000
NODE_ENV=production
ALLOWED_ORIGINS=https://your-user-frontend.pages.dev,https://your-admin-frontend.pages.dev
```

## 二、用户端前端环境变量

在项目根目录创建 `.env.production` 文件：

```env
VITE_API_BASE_URL=https://your-backend.railway.app/api
```

### Cloudflare Pages环境变量

在Cloudflare Pages项目设置中添加：

```
VITE_API_BASE_URL = https://your-backend.railway.app/api
```

## 三、后台管理前端环境变量

在 `admin/` 目录创建 `.env.production` 文件：

```env
VITE_API_BASE_URL=https://your-backend.railway.app/api
```

### Cloudflare Pages环境变量

在Cloudflare Pages项目设置中添加：

```
VITE_API_BASE_URL = https://your-backend.railway.app/api
```

## 四、本地开发环境变量

### 用户端前端
在项目根目录创建 `.env.local`：

```env
VITE_API_BASE_URL=http://localhost:8888/api
```

### 后台管理前端
在 `admin/` 目录创建 `.env.local`：

```env
VITE_API_BASE_URL=http://localhost:8888/api
```
