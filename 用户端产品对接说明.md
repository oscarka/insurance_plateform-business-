## 二、数据流程

```
后台管理系统
  ↓ 配置产品（状态：启用）
  ↓ 保存到数据库
  ↓
后端API (8888)
  ↓ 查询启用的产品
  ↓ 返回：{ success: true, data: [...] }
  ↓
用户端 (5173)
  ↓ 调用 getProducts()
  ↓ 显示产品列表
```

## 三、验证步骤

### 步骤1：在后台管理系统配置产品

1. 访问：http://localhost:5174/admin/products
2. 确保产品状态为"启用"
3. 如果没有产品，点击"+ 添加产品"创建

### 步骤2：验证后端API

```bash
# 测试产品列表API
curl http://localhost:8888/api/products

# 应该返回：
# {
#   "success": true,
#   "data": [
#     {
#       "product_id": 1,
#       "product_name": "雇主责任险A",
#       "status": "启用",
#       ...
#     }
#   ],
#   "count": 1
# }
```

### 步骤3：在用户端查看

1. 访问：http://localhost:5173/#/new-policy
2. 应该能看到后台配置的启用产品
3. 选择产品后，应该能看到该产品的方案列表

## 四、注意事项

### 1. 产品状态
- 只有状态为"启用"的产品才会显示在用户端
- 后台可以设置产品状态为"禁用"，禁用后用户端不会显示

### 2. API地址
- 用户端API地址：`http://localhost:8888/api`
- 确保后端服务运行在8888端口

### 3. CORS配置
- 后端已配置允许所有来源的CORS
- 如果遇到跨域问题，检查后端CORS配置

## 五、故障排查

### 问题1：用户端看不到产品

**检查**：
1. 后端服务是否启动：`curl http://localhost:8888/health`
2. 产品是否启用：在后台管理系统查看产品状态
3. 浏览器控制台是否有错误
4. 网络请求是否成功（F12 → Network）

**解决**：
- 确保后端服务运行
- 确保产品状态为"启用"
- 检查API地址配置

### 问题2：API返回空数组

**检查**：
1. 数据库中是否有启用的产品
2. SQL查询是否正确

**解决**：
```sql
-- 检查产品状态
SELECT product_id, product_name, status 
FROM insurance_products 
WHERE status = '启用';
```

### 问题3：跨域错误

**检查**：
- 浏览器控制台是否有CORS错误

**解决**：
- 检查后端CORS配置
- 确保API地址正确
